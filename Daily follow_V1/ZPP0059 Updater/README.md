# ZPP0059 Auto-Updater — ดึง ZPP0059 เข้า sqlite อัตโนมัติ

โปรแกรมเล็ก ๆ ที่รัน **แยกจาก** `serve_daily_follow.py` โดยจะสั่ง SAP รัน ZPP0059
เป็นรอบ ๆ (ค่าเริ่มต้นทุก ~2 นาที) แล้วเขียนยอดผลิตล่าสุดลง `zpp0059.db` ตัวเดิม
ที่หน้า Daily Follow ใช้อยู่ → หน้า dashboard จะเห็นค่า **Actual unit rate** อัปเดต
เองโดยไม่ต้องกดปุ่มเขียว

## หลักการ
- **ไม่ก๊อปโค้ด** — `update_zpp0059.py` import ฟังก์ชันเดิมจาก `serve_daily_follow.py`
  (`run_zpp0059`) มาเรียกวนใน loop จึงใช้ตรรกะขับ SAP ชุดเดียวกันเป๊ะ
- **gap-based** — ดึงเสร็จ → พัก N วินาที → ดึงใหม่ ไม่ซ้อนกันไม่ว่า SAP จะช้าแค่ไหน
- **ดึงซ้ำไม่เกิดข้อมูลซ้ำ** — `db_upsert` ใช้ business key
  (Order + Activity + Short Time Stamp + Tag ID) ของเก่าเขียนทับ ของใหม่เพิ่มเข้า

## ทำไมไม่ชนกับหน้าเว็บ
- **SAP** : ทั้งปุ่มเขียว, ปุ่ม ZPP0022 และตัวนี้ วิ่งผ่าน `_drive_sap_export` ซึ่งถือ
  file-lock กลาง (`_sap_automation.lock` ในโฟลเดอร์ Daily follow) → รันได้ทีละตัว
  ถ้าชนกันตัวที่มาทีหลังจะรอ/ข้ามรอบ ไม่แย่งหน้าต่าง SAP กัน
- **DB** : `_get_db` เปิด `journal_mode=WAL` + `busy_timeout` → หน้าเว็บ (อ่าน)
  กับตัวนี้ (เขียน) ทำงานพร้อมกันบนไฟล์เดียวได้อย่างปลอดภัย

> ⚠️ SAP GUI Scripting ใช้ **หน้าต่าง SAP เดียวกับที่คนนั่งใช้** ถ้าดึงถี่ทุก 2 นาที
> ควรรันบน **เครื่อง/login ที่ไม่มีคนใช้ SAP ทำอย่างอื่น** (หรือยอมรับว่าห้ามแตะ SAP
> ตอนโปรแกรมกำลังดึง)

## สิ่งที่ต้องมี
- Windows + **Python** (ชุดเดียวกับที่รัน Daily Follow)
- **SAP GUI** เปิดและ login ค้างไว้ + เปิด **SAP GUI Scripting**
- โฟลเดอร์นี้ต้องวางไว้ **ข้าง ๆ** โฟลเดอร์ `Daily follow` (โครงสร้าง `Daily follow_V1/`)
- ต้องเข้าถึง `zpp0059.db` ตัวเดียวกับหน้าเว็บได้ → รันบนเครื่องเดียวกับที่เก็บ DB

## วิธีใช้
1. ดับเบิลคลิก **`Start_ZPP0059_Updater.bat`**
   (หรือสั่ง `python update_zpp0059.py`)
2. โปรแกรมจะดึง ZPP0059 ทุก ~2 นาที และ log ผลแต่ละรอบ เช่น
   ```
   [2026-07-01 09:30:12] OK  EXPORT_xxx.xlsx · +18 ใหม่ / 5 อัปเดต · DB รวม 12,840 แถว · 22.4s
   ```
3. กด **Ctrl+C** เพื่อหยุด

## ตัวเลือก
- `--interval N` — พักหลังดึงเสร็จ N วินาที (ขั้นต่ำ 60, ค่าเริ่มต้น 120)
  เช่น `python update_zpp0059.py --interval 90`
- `--once` — ดึงครั้งเดียวแล้วออก (ไว้ทดสอบว่าต่อ SAP ได้ไหม)

## เลือก interval เท่าไรดี
1 รอบใช้เวลาราว **15–60 วินาที** (ขึ้นกับ SAP/เน็ตเวิร์ก) — ตั้งต่ำกว่านี้ไม่มีความหมาย
เพราะรอบยังไม่เสร็จ แนะนำ **90–120 วินาที** เพราะข้อมูล ZPP0059 เกิดตอนพนักงาน
post งานเป็นล็อต ไม่ได้เปลี่ยนทุกวินาที ดึงทุก 2 นาทีก็สดพอและไม่เป็นภาระ SAP

หน้าเว็บฝั่ง badge จะ poll ข้อมูลใหม่ทุก 60 วินาทีอยู่แล้ว จึงเห็นค่าที่อัปเดตภายใน ~1 นาที
