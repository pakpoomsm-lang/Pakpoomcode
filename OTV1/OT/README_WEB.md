# ระบบบันทึกการทำ OT พนักงาน (Web Application)
## Heat Exchange Indoor

## 🌐 คำอธิบาย
โปรแกรมเว็บแอปพลิเคชันสำหรับบันทึกและจัดการข้อมูลการทำงานล่วงเวลา (OT) ของพนักงาน
พัฒนาด้วย **Python Flask** + **HTML** + **Bootstrap 5** + **JavaScript**

---

## ✨ คุณสมบัติหลัก

### 1. 📝 บันทึกการทำ OT
- เลือกวันที่ทำงานด้วย Date Picker
- กรองพนักงานตาม GL, Process, และ Shift
- เพิ่มรายการงานพร้อมรายละเอียด
- กำหนดสถานะ (NT, Leave, MCP, Subcon)
- Preview ข้อมูลก่อนบันทึก
- บันทึกข้อมูลลงฐานข้อมูล SQLite

### 2. 📊 รายงาน OT
- กรองข้อมูลตามช่วงวันที่
- กรองตาม Process
- แสดงผลรายงานในรูปแบบตาราง
- Export ข้อมูลเป็นไฟล์ Excel (.xlsx)
- ดาวน์โหลดรายงานได้ทันที

### 3. 👥 จัดการพนักงาน
- นำเข้าข้อมูลพนักงานจากไฟล์ Excel
- แสดงรายชื่อพนักงานทั้งหมด (130 คน)
- ค้นหาพนักงานแบบ Real-time
- ลบและจัดการข้อมูลพนักงาน

## 🚀 การติดตั้งและใช้งาน

### ขั้นตอนที่ 1: ติดตั้ง Python Libraries

```bash
pip install flask pandas openpyxl
```

### ขั้นตอนที่ 2: เริ่มต้นใช้งาน

**วิธีที่ 1: ใช้ไฟล์ .bat (แนะนำ)**
```
Double-click ที่ไฟล์ run_web.bat
```

**วิธีที่ 2: ใช้ Command Line**
```bash
python app.py
```

เปิดเว็บเบราว์เซอร์ (Chrome, Edge, Firefox) แล้วไปที่:
```
http://localhost:5000
```

## 📖 คู่มือการใช้งาน

### 1. นำเข้าข้อมูลพนักงาน (ครั้งแรก)

1. คลิกที่เมนู **"จัดการพนักงาน"**
2. คลิกปุ่ม **"เลือกไฟล์ Excel"**
3. เลือกไฟล์ `รายชื่อพนักงาน 05_2026.xlsx`
4. คลิกปุ่ม **"นำเข้าข้อมูล"**
5. รอจนกว่าจะแสดงข้อความสำเร็จ

### 2. บันทึกการทำ OT

1. คลิกที่เมนู **"บันทึก OT"**
2. **เลือกวันที่ทำงาน**
3. **กรองพนักงาน:**
   - เลือก GL (หัวหน้างาน)
   - เลือก Process (สถานที่ทำงาน)
   - เลือก Shift (A/B/D)
4. **เพิ่มรายการงาน:**
   - คลิกปุ่ม **+** ที่แถวของพนักงาน
   - กรอกรายการงาน
   - กรอก Remark (ถ้ามี)
   - เลือกสถานะ
   - คลิก **"บันทึก"**
5. **ทำซ้ำ** ขั้นตอนที่ 4 สำหรับพนักงานคนอื่นๆ
6. **ตรวจสอบข้อมูล:**
   - คลิกปุ่ม **"Preview"** เพื่อดูภาพรวม
7. **บันทึกข้อมูล:**
   - คลิกปุ่ม **"บันทึกข้อมูล"**
   - ยืนยันการบันทึก

### 3. ดูรายงาน OT

1. คลิกที่เมนู **"รายงาน"**
2. **เลือกช่วงวันที่:**
   - จากวันที่
   - ถึงวันที่
3. **เลือก Process** (หรือเลือก "All")
4. คลิกปุ่ม **"ค้นหา"**
5. **Export Excel (ถ้าต้องการ):**
   - คลิกปุ่ม **"Export Excel"**
   - ไฟล์จะดาวน์โหลดอัตโนมัติ

### 4. จัดการข้อมูลพนักงาน

1. คลิกที่เมนู **"จัดการพนักงาน"**
2. **ค้นหาพนักงาน:**
   - พิมพ์ในช่องค้นหา (รหัส, ชื่อ, สถานที่ทำงาน)
3. **นำเข้าข้อมูลใหม่:**
   - คลิก "เลือกไฟล์ Excel" → "นำเข้าข้อมูล"
4. **ลบข้อมูล:**
   - คลิก "ลบข้อมูลพนักงานทั้งหมด" (ข้อมูล OT จะยังคงอยู่)

## 🗄️ ฐานข้อมูล

โปรแกรมใช้ **SQLite** เป็นฐานข้อมูล (ไฟล์ `ot_records.db`)

### ตาราง employees
```sql
- emp_id          รหัสพนักงาน
- first_name      ชื่อ
- last_name       นามสกุล
- workplace       สถานที่ทำงาน
- group_name      กลุ่มงาน (Group)
- emp_type        ประเภท (MCP/SUB)
- shift           กะ (A/B/D)
- supervisor      หัวหน้างาน/GL
```

### ตาราง ot_records
```sql
- work_date       วันที่ทำงาน
- emp_id          รหัสพนักงาน
- workplace       สถานที่ทำงาน
- work_description รายการงาน
- remark          หมายเหตุ
- status          สถานะ (NT/Leave/MCP/Subcon)
- created_at      วันเวลาที่บันทึก
```

## 📊 API Endpoints

### Employees
- `GET /api/supervisors` - รายชื่อ GL/หัวหน้างาน
- `GET /api/workplaces` - รายชื่อสถานที่ทำงาน
- `GET /api/employees/filter` - กรองพนักงาน
- `GET /api/employees/list` - รายชื่อพนักงานทั้งหมด
- `GET /api/employees/count` - จำนวนพนักงาน
- `POST /api/employees/import` - นำเข้าข้อมูลพนักงาน
- `POST /api/employees/clear` - ลบข้อมูลพนักงาน

### OT Records
- `POST /api/ot/save` - บันทึก OT
- `GET /api/reports/search` - ค้นหารายงาน
- `GET /api/reports/export` - Export Excel

## 🔧 การแก้ไขปัญหา

### ❌ ปัญหา: เซิร์ฟเวอร์ไม่เริ่มต้น
**แก้ไข:**
```bash
pip install flask pandas openpyxl
```

### ❌ ปัญหา: เข้าถึง http://localhost:5000 ไม่ได้
**แก้ไข:**
- ตรวจสอบว่าเซิร์ฟเวอร์กำลังรันอยู่
- ลองเปลี่ยนเป็น http://127.0.0.1:5000

### ❌ ปัญหา: ไม่สามารถนำเข้าข้อมูลพนักงาน
**แก้ไข:**
- ตรวจสอบว่าไฟล์ Excel มีโครงสร้างถูกต้อง
- ดูตัวอย่างไฟล์ `รายชื่อพนักงาน 05_2026.xlsx`

### ❌ ปัญหา: ข้อมูลไม่แสดงในรายงาน
**แก้ไข:**
- ตรวจสอบช่วงวันที่ที่เลือก
- ตรวจสอบว่ามีการบันทึกข้อมูลในช่วงนั้นจริง

### ❌ ปัญหา: Port 5000 ถูกใช้งานอยู่
**แก้ไข:**
- เปลี่ยน port ในไฟล์ `app.py` (บรรทัดสุดท้าย)
```python
app.run(debug=True, host='0.0.0.0', port=5001)
```

## 🔐 การใช้งานในเครือข่าย (LAN)

หากต้องการให้เครื่องอื่นในเครือข่ายเข้าถึงได้:

1. เช็ค IP Address ของเครื่อง Server:
```bash
ipconfig  # Windows
ifconfig  # Linux/Mac
```

2. เครื่องอื่นเข้าถึงผ่าน:
```
http://<IP-Address>:5000
```
ตัวอย่าง: `http://192.168.1.100:5000`

**หมายเหตุ:** ตรวจสอบ Firewall ว่าอนุญาต port 5000
