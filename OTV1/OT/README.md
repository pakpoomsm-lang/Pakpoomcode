# ระบบบันทึกการทำ OT พนักงาน - Heat Exchange Indoor

## คำอธิบาย
โปรแกรมสำหรับบันทึกและจัดการข้อมูลการทำงานล่วงเวลา (OT) ของพนักงาน พัฒนาด้วย Python และ Tkinter

## คุณสมบัติหลัก

### 1. บันทึกการทำ OT
- เลือกวันที่ทำงาน
- กรองพนักงานตาม GL, Process, และ Shift
- เพิ่มรายการงานสำหรับแต่ละพนักงาน
- ระบุรายละเอียดงานและ Remark
- กำหนดสถานะ (NT, Leave, MCP, Subcon)
- Preview ข้อมูลก่อนบันทึก
- บันทึกข้อมูลลงฐานข้อมูล SQLite

### 2. รายงาน OT
- กรองข้อมูลตามช่วงวันที่
- กรองตาม Process
- แสดงผลรายงานในตาราง
- Export ข้อมูลเป็นไฟล์ Excel

### 3. จัดการพนักงาน
- นำเข้าข้อมูลพนักงานจากไฟล์ Excel
- แสดงรายชื่อพนักงานทั้งหมดในระบบ
- ลบข้อมูลพนักงาน

## การติดตั้ง

### ติดตั้ง Python Libraries ที่จำเป็น
```bash
pip install pandas openpyxl tkcalendar
```

### ไฟล์ที่จำเป็น
- `ot_recording_system.py` - โปรแกรมหลัก
- `รายชื่อพนักงาน 05_2026.xlsx` - ไฟล์ข้อมูลพนักงาน

## วิธีการใช้งาน

### 1. เริ่มต้นใช้งานครั้งแรก

1. รันโปรแกรม:
```bash
python ot_recording_system.py
```

2. ไปที่แท็บ "จัดการพนักงาน"

3. คลิกปุ่ม "เลือกไฟล์ Excel" และเลือกไฟล์ `รายชื่อพนักงาน 05_2026.xlsx`

4. โปรแกรมจะนำเข้าข้อมูลพนักงานทั้งหมดเข้าสู่ระบบ

### 2. บันทึกการทำ OT

1. ไปที่แท็บ "บันทึกการทำ OT"

2. เลือกวันที่ทำงาน

3. เลือก GL หรือ Process และ Shift ที่ต้องการ (หรือเลือก "All" เพื่อแสดงทั้งหมด)

4. Double-click ที่ชื่อพนักงานที่ต้องการบันทึก OT

5. ในหน้าต่างป๊อปอัพ:
   - ระบุรายการงาน
   - ระบุ Remark (ถ้ามี)
   - เลือกสถานะ (NT, Leave, MCP, Subcon)
   - คลิก "บันทึก"

6. ทำซ้ำขั้นตอนที่ 4-5 สำหรับพนักงานคนอื่นๆ

7. เมื่อบันทึกรายการงานครบแล้ว:
   - คลิก "Preview" เพื่อตรวจสอบข้อมูล
   - คลิก "บันทึกข้อมูล" เพื่อบันทึกลงฐานข้อมูล

8. คลิก "ล้างข้อมูล" เพื่อล้างรายการที่บันทึกไปแล้ว

### 3. ดูรายงาน OT

1. ไปที่แท็บ "รายงาน OT"

2. เลือกช่วงวันที่ที่ต้องการดูรายงาน
   - จากวันที่
   - ถึงวันที่

3. เลือก Process (หรือเลือก "All" เพื่อแสดงทั้งหมด)

4. คลิก "ค้นหา"

5. ระบบจะแสดงรายงาน OT ตามเงื่อนไขที่เลือก

6. หากต้องการ Export ข้อมูล:
   - คลิก "Export Excel"
   - เลือกตำแหน่งที่จะบันทึกไฟล์
   - ระบบจะสร้างไฟล์ Excel ที่สามารถนำไปใช้งานต่อได้

### 4. จัดการข้อมูลพนักงาน

1. ไปที่แท็บ "จัดการพนักงาน"

2. ดูรายชื่อพนักงานทั้งหมดในระบบ

3. หากต้องการนำเข้าข้อมูลพนักงานใหม่:
   - คลิก "เลือกไฟล์ Excel"
   - เลือกไฟล์ Excel ที่มีโครงสร้างเหมือนกับไฟล์ตัวอย่าง

4. หากต้องการลบข้อมูลพนักงานทั้งหมด:
   - คลิก "ลบข้อมูลพนักงานทั้งหมด"
   - ยืนยันการลบ
   - (หมายเหตุ: ข้อมูล OT จะยังคงอยู่ในระบบ)

## โครงสร้างไฟล์ Excel สำหรับนำเข้าพนักงาน

ไฟล์ Excel ต้องมีคอลัมน์ดังนี้ (ตามลำดับ):

| คอลัมน์ที่ | ชื่อคอลัมน์ | คำอธิบาย |
|-----------|------------|---------|
| 0 | สถานที่ทำงาน | สถานที่/เครื่องจักรที่ทำงาน |
| 1 | รหัส | รหัสพนักงาน |
| 2 | ชื่อ | ชื่อพนักงาน |
| 3 | นามสกุล | นามสกุลพนักงาน |
| 4 | Type | ประเภทพนักงาน (MCP/SUB) |
| 5 | SHIFT | กะการทำงาน (A/B/D) |
| 6 | หัวหน้างาน | ชื่อหัวหน้างาน/GL |
| 10 | GL | หมายเลข GL |

## ฐานข้อมูล

โปรแกรมใช้ SQLite เป็นฐานข้อมูล โดยจะสร้างไฟล์ `ot_records.db` ในโฟลเดอร์เดียวกับโปรแกรม

### ตาราง employees
เก็บข้อมูลพนักงาน
- emp_id (รหัสพนักงาน)
- first_name (ชื่อ)
- last_name (นามสกุล)
- workplace (สถานที่ทำงาน)
- emp_type (ประเภทพนักงาน)
- shift (กะการทำงาน)
- supervisor (หัวหน้างาน)
- gl_number (หมายเลข GL)

### ตาราง ot_records
เก็บข้อมูลการทำ OT
- work_date (วันที่ทำงาน)
- emp_id (รหัสพนักงาน)
- workplace (สถานที่ทำงาน)
- work_description (รายการงาน)
- remark (หมายเหตุ)
- status (สถานะ: NT/Leave/MCP/Subcon)
- created_at (วันเวลาที่บันทึก)

## คำอธิบายสถานะ
- **NT** = Normal Time (เวลาปกติ)
- **Leave** = ลา
- **MCP** = พนักงาน MCP
- **Subcon** = พนักงาน Subcontract

## ข้อควรระวัง
1. ข้อมูลพนักงานจะถูกแทนที่ทุกครั้งที่นำเข้าใหม่
2. ข้อมูล OT จะไม่ถูกลบแม้จะลบข้อมูลพนักงาน
3. สำรองฐานข้อมูล `ot_records.db` เป็นประจำ
4. ไฟล์ Excel ที่ Export ออกมาสามารถนำไปประมวลผลต่อได้

## การแก้ไขปัญหา

### ปัญหา: โปรแกรมไม่สามารถเริ่มต้นได้

- ตรวจสอบช่วงวันที่ที่เลือก
- ตรวจสอบว่ามีการบันทึกข้อมูลในช่วงวันที่นั้นจริง
